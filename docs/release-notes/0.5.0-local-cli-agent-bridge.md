# 更新说明 · 0.5.0

## 新功能：在飞书上接管本地 Claude Code / Codex

以前只能在飞书里指挥本机干活。这一版反过来：你在电脑终端里直接运行的 **Claude Code / Codex**，现在也能把关键节点推到**你的飞书私聊**，让你人不在电脑前也能继续推进。

开启后，本地 agent 在这些时刻会给你发飞书卡片：

- **要权限时**：弹出审批卡，你可以「批准一次」「本会话放行」或「拒绝」。
- **向你提问时**（Claude 的选择式提问）：直接在卡片上点选项，或填写自定义回答。
- **任务做完时**：推送完成通知和最终回答摘要；你还能**直接回复这张卡**，把消息接着喂给本地 agent 继续聊。

## 默认只在你离开时打扰

默认是「**离开才转发**」：只有当电脑闲置超过一段时间，才会把上面的卡片发到飞书。

## 怎么开启

机器人私聊 → 打开**设置卡** → 找到「🖥️ 本地 agent」：

1. 打开「飞书接管本地 agent」总开关；
2. 点一下「修复 hooks」，让 Claude Code 和 Codex 接上；
3. 看到两者状态都变成「已安装」即可。

> 开启前需要先设置机器人 owner（转发的目标就是 owner 的私聊）。

## 平台说明

- **macOS**：完整可用。
- **Windows**：可用，但「离开检测」为实验性；万一检测不准，会选择照常转发，不会漏掉通知。

## 隐私

- 默认**不**记录对话原文，转发目标只有机器人 owner 本人的私聊。
- 桥接自己在飞书里发起的会话默认不会再转发回来，避免重复打扰。
